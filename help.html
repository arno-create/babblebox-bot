Hosted help page

Babblebox, explained clearly.

Babblebox is built around a few clean lanes instead of a giant command maze: party games when the room is active, anonymous confessions when a community needs staff-blind posting, Daily Arcade for lighter rituals, identity surfaces worth showing, utilities that stay quiet, and compact admin safety when a server needs it.

This page covers setup, what stays public or private, the commands people actually use, and how Shield behaves without pretending every heuristic is certain.

Public-friendly

Public-friendly surfaces usually lean public. Sensitive setup, reminders, and DM alert tools lean private.

Start fast

Invite the bot, use /help or /support, then try /play, /drops status, /daily, or /watch.

Shield stays bounded

Shield live moderation is optional, posture-aware, and built around compact moderator alerts instead of a huge archive. Bounded feature-surface checks still protect Confessions links, AFK, reminders, and watch keywords privately.

Quick start

  1. Invite Babblebox and make sure it can send embeds where you want people to use it.
  2. Use /help or /support in Discord, or jump through the sections below.
  3. Adopt one lane first: /play, /drops status, /daily, /buddy, or /watch.
  4. Turn on Shield live moderation only if your server actually needs chat automation; bounded feature-surface safety still applies privately where it matters.
Jump straight to the lane you need

This guide is organized by actual product lanes, not by every command in alphabetical order. If you are onboarding a server, start with one lane, get it working in one channel, and then expand from there.

Getting Started

Start with one lane, not every command.

Party servers usually start with /play and /daily. Retention-focused communities often add /drops status and scheduled Question Drops next. Smaller communities often start with /watch, /later, or /remind. Identity-heavy servers tend to start with /buddy and /profile.

Best first public command

/play opens the multiplayer lobby for Broken Telephone, Exquisite Corpse, Spyfall, Word Bomb, and Pattern Hunt.

Best first solo command

/daily opens Daily Arcade without needing a full room.

Best first private utility

/watch settings is a good first setup if you want quiet DM alerts instead of noisy public posts.

Best first admin command

/drops status, /shield panel, or /admin panel gives moderators compact control surfaces first.

Best first support command

/support opens the official support server, GitHub repository, and website links in one place if you need help or want to report a bug.

Public vs Private

Showable where it should be. Quiet where it should stay personal.

Babblebox separates surfaces people want to share from surfaces that work better as personal tools. That keeps the bot feeling native in Discord instead of turning every interaction into a private panel or every reminder into channel clutter.

Usually public

/play, most Daily Arcade outputs, /buddy, and /profile are designed to sit comfortably in chat.

Usually private

/watch, /later, /capture, /remind, /afk, and most setup or failure states stay private or DM-oriented.

Moderator-private

Shield configuration, admin cleanup tooling, and mod alerts are meant for admins and moderator channels, not public callouts.

Mobile-friendly intent

Cards aim for short titles, compact lists, and spacing that still scans cleanly on Discord mobile.

Anonymous Confessions

Staff-blind anonymous posting with calm, bounded safety.

Confessions are optional in Babblebox and do nothing until admins enable and configure them. Once live, /confess opens a private composer, /confess manage, /confess appeal, /confess report, /confess reply-to-user, and /confess about mirror the private member utility flows, and server staff still moderate by confession ID and case ID only while Babblebox still enforces safety internally without exposing the author.

/confess plus manage, appeal, report, and about

Open the private composer or the matching private member support flows.

Confessions, anonymous replies, owner replies, and pending self-edits allow up to 4000 characters. One link total is allowed under the server's active Confessions link mode, and Babblebox Shield still blocks unsafe or high-risk destinations. Offensive or derogatory language is filtered, adult / 18+ language is blocked by default unless admins change that policy, images stay off by default, public anonymous replies stay off by default, and self-edit stays off by default unless admins explicitly enable those riskier options. When public replies are enabled, the button is named Reply anonymously, it lives on eligible published confession posts instead of the launch panel, first replies to top-level confessions reuse one Babblebox thread when Discord allows it, reply-confessions stay inside that same thread without nested threads, and the reply stays anonymous even if Babblebox sends it through private approval before posting. The latest eligible live confession post can also show Create a confession. Babblebox hides your account, but a personal link or identifiable image can still reveal you if you include it.
/confess reply-to-user

Review private owner reply prompts and post an anonymous owner reply publicly.

If someone explicitly replies to your confession or your first public owner reply, Babblebox can DM you a compact owner prompt. The same owner reply inbox is always available privately through this command, it only tracks explicit Discord replies instead of loose same-channel chatter, and the public result posts as Anonymous Owner Reply. Owner replies are enabled by default, stay text-only, and only go through review if admins enable owner-reply review.
/confessions, /confessions role, and /confessions moderate

Open the admin panel, manage role eligibility, and moderate by confession ID or case ID.

Admins can enable the feature, choose `Disabled`, `Trusted Only`, or `Allow All Safe` Confessions link mode, explicitly turn images or replies on after setting a separate private review channel, configure a dedicated appeals / reports channel, and manage role allowlist / blacklist rules where blacklist always wins and a non-empty allowlist means only those roles may submit. Babblebox shows a warning before `Allow All Safe` because Shield safety still applies but wider link acceptance increases moderation and privacy risk. Staff can still approve, deny, delete, pause, restrict images, clear, or mark false positive without exposing the author.
Party Games

Short multiplayer energy, built around live rooms.

/play acts as the shared lobby for the live-party lane instead of forcing people to remember several different entry points.

/play

Open the multiplayer lobby.

Start Broken Telephone, Exquisite Corpse, Spyfall, Word Bomb, or Pattern Hunt from one place. Broken Telephone runs as a DM relay, Exquisite Corpse hides every prompt until the stitched reveal, Spyfall rotates the spotlight through live interrogation and voting, Word Bomb keeps the syllable pressure high, and Pattern Hunt keeps private rule guesses in /hunt guess. Coders need server DMs open before the room starts.
/hunt status and /hunt guess

Operate the active Pattern Hunt round.

These stay dormant unless a Pattern Hunt lobby is already live in the server. /hunt status mirrors the live state card privately, and /hunt guess is built for 1-3 clean rule families that stay private while the public chat keeps reading the clues. The Contains Digits guess family means digits 0-9 only, and slash is the cleanest way to submit multi-family theories.
/stats and /leaderboard

Check light session stats.

These stay compact and playful instead of turning into a giant progression system.
Daily Arcade

Three quick daily booths plus a guild-first Question Drops lane.

Daily Arcade gives people something to do even when there is no full group ready for a live room. Question Drops add lightweight scheduled prompts, mastery roles, scholar ranks, guild knowledge boards, and private member role controls without turning the server into a spammy trivia cannon. Both lanes stay compact, public-friendly, and spoiler-aware.

/daily

Open the current daily booth selection.

Babblebox keeps Daily Arcade at three booths: Shuffle leans on stronger word shapes, Emoji uses layered clue trails, and Signal rotates across Caesar shift, mirror alphabet, and adjacent-pair swap.
/daily play

Submit a daily answer or guess.

The shorthand path still defaults cleanly instead of forcing command complexity.
/daily stats, /daily share, /daily leaderboard

Share progress without turning the booth into a spoiler mess.

Open and result cards show Difficulty, Length, and the booth Profile, while warnings and storage issues stay private even when the main cards are public.
/drops status, /dropsadmin config, /dropsadmin channels, /dropsadmin categories, /drops stats, /drops leaderboard

Run the scheduled Question Drops lane.

Admins can run 1-10 drops per day, tune activity gating, and choose channels, categories, tone, and a compact difficulty profile. Standard stays welcoming, Smart leans medium and hard, Hard is noticeably tougher, players still answer live in-channel, numeric drops accept clean digits, and simple number words only count for whole-number answers, multiple-choice prompts accept the correct option letter or option text, ordered-sequence prompts are one-shot and work best with a clean comma-separated answer, idle channels can skip a slot, higher daily counts reuse concepts sooner, and a live drop blocks new party rooms in that same channel until it resolves.
/drops roles status, /drops roles remove, /drops roles preference

Manage your own Babblebox Question Drops roles privately.

Members can remove current Babblebox-managed roles, stop future grants, or opt back in later without erasing earned knowledge history.
/dropsadmin mastery category, /dropsadmin mastery scholar, /dropsadmin mastery recalc

Configure mastery roles, the scholar ladder, and announcement copy in one place.

Those same mastery commands open the safe modal editor with template_action, support a scope default plus optional tier overrides, and fall back as tier override > scope default > Babblebox default. Category tokens: {user.mention} {user.name} {user.display_name} {role.name} {tier.label} {threshold} {category.name}. Scholar tokens: {user.mention} {user.name} {user.display_name} {role.name} {tier.label} {threshold}.
Buddy / Profile / Vault

One identity layer, not a bloated economy.

Babblebox's identity surfaces are meant to feel worth showing. They collect your ongoing flavor, streaks, achievements, party-game wins, Question Drops progress, mastery flavor, and scholar rank into a few readable surfaces rather than exploding into a giant inventory game.

/buddy

Your public-friendly mascot or companion surface.

Good for quick status, style, and personality without a wall of admin text.
/profile

Your broader public identity card.

Useful when you want a showable summary that still feels readable in a real Discord channel, including guild-first knowledge lane progress without confusing it with Daily Arcade.
/vault

Your more personal storage or progression surface.

Vault is the less public lane when you want to manage your own state quietly.
Utilities

Quiet tools for real server life.

Utilities are where Babblebox earns its keep on ordinary days. They reduce missed context and small task friction without filling channels with bot chatter.

/watch mentions, /watch replies, /watch keyword

Get private DM alerts for the things you care about.

Watch is built for low-noise monitoring, not public surveillance theater, and keyword setup keeps local validation plus a narrow privacy-only Shield safety check.
/later mark

Save a place in a server so you can come back later.

Useful when you want to keep momentum without opening tabs or pinning messages everywhere.
/capture

Send yourself a private recent-message snapshot.

Good when you want a clean recap with message context and a jump-back link, without turning the server into a scrapbook.
/remind

Set reminders without needing a separate productivity bot.

Reminder setup and failure states stay private so channels do not fill with housekeeping, and Babblebox rechecks public reminder delivery through private Shield privacy, adult, and severe checks before posting it later.
/afk, /afktimezone, /afkschedule

Set temporary AFK state, timezone hints, and repeating schedules.

This helps Babblebox explain absence cleanly without forcing manual status updates every time, while AFK reasons stay on a bounded private Shield lane for privacy, adult, and severe checks.
Shield / Admin

Compact safety for moderators, with honest confidence.

Shield live moderation is optional and starts from local rules. It can look for likely privacy leaks, invite or promo abuse, a first-class Anti-Spam lane grounded in explicit message-rate and near-duplicate thresholds, a separate GIF Flood / Media Pressure lane for repeated low-text GIF disruption, malicious or scam links, no-link DM-lure scams that route nitro, crypto, prize, or cash bait into DMs or off-platform contact, adult-domain intel, an optional conservative solicitation / DM-ad detector inside the Adult Links + Solicitation pack, and a separate Severe Harm / Hate pack for sexual-exploitation solicitation, explicit self-harm encouragement, eliminationist or dehumanizing hate, and severe slur abuse with quote, reporting, moderation, and rules-context suppressors. It can also rescan meaningful edits and webhook or community-post style deliveries in scope, but it is designed to stay conservative when the signal is weak, especially for ordinary bot or webhook posts that only carry medium or soft scam heuristics. Attachment filenames stay metadata-only for link repetition unless a real link was actually posted. Shield also has a separate Trusted Links Only live-message mode for servers that want the built-in trusted pack plus bounded domain or invite allowlists only, and that mode stays separate from Confessions link mode. /shield trusted now exposes the built-in trusted families, direct domains, and any local built-in trust disables. Specific adult-topic channels can relax only the optional solicitation detector without weakening adult-domain, scam, malicious-link, or trusted-brand impersonation protections. The premium configuration path is /shield panel: pick a pack, then use Actions, Options, or Exemptions so unrelated spam or GIF controls never spill into Severe or Hate work, while each pack can inherit the global timeout or keep a dedicated timeout profile. Safe mainstream domains are bypassed before link suspicion, local malicious-link checks include a feed of about 200k known malicious domains, and phase 1 does not depend on any outside reputation provider. Trusted-brand impersonation and lookalike phishing hosts are also hard-blocked locally. Unknown risky links only escalate after combined local scam evidence. The first live enable applies a recommended non-AI baseline, while Anti-Spam stays local-first with short-lived windows plus bounded corroboration so healthy active chat stays safer while real spam can still log, delete, timeout, or combine delete + timeout when the configured evidence is strong enough. Optional emote and capitals lanes stay off until an admin enables them, moderators are exempt from Anti-Spam by default unless an admin deliberately chooses delete-only or full enforcement, and pack-specific exemptions let one channel, role, or member bypass one pack without weakening the rest. The separate GIF Flood / Media Pressure pack keeps GIF disruption legible instead of burying it under generic spam, keeps grouped incidents per offender and incident window for one-user floods, splits one-user GIF floods from collective channel pressure, groups collective pressure per channel, deletes the matched burst for one-user abuse, removes the exact live GIF streak for true consecutive streak floods, trims only the newest contributing GIF posts from the active pressure slice for channel-pressure matches after lightweight meaningful-text weighting, can combine shared cleanup with one member's personal enforcement in mixed incidents, never touches healthy text, and now exposes tighter low-end controls for stricter rooms without turning the panel into expert-only clutter. The linked Admin lane focuses on one direct emergency lock lane plus follow-up, exclusions, logs, and truthful permission diagnostics. /lock channel and /lock remove give moderators who can manage channels or messages, timeout, kick, or ban members, plus admins one calm direct moderation lane by default, /lock settings lets admins tune the default notice and optionally make that lane admin-only, /timeout remove gives moderators a direct timeout-clear lane, and /admin permissions explains which missing bot permissions degrade locks, timeout removal, Shield actions, or role-based helpers. Separate from that live lane, Shield also supplies private bounded feature-surface checks for Confessions unsafe-link parity, AFK reasons, reminder text plus public reminder delivery, and watch keyword setup. AFK reasons and reminders use privacy, adult, and severe checks there; watch keyword setup stays privacy-only. Those private feature checks do not create mod-log alerts and do not call Shield AI.

Main admin commands

/shield panel, /shield module, /shield escalation, /shield rules, /shield links, /shield trusted, /shield severe category, /shield severe term, /shield logs, /shield filters, /shield exemptions, /shield allowlist, /shield ai, /shield test, /lock channel, /lock remove, /lock settings, /timeout remove, /admin panel, /admin followup, /admin logs, /admin exclusions, and /admin permissions.

Graduated local policy

Shield uses Low confidence, Medium confidence, and High confidence actions. Low confidence stays safest, Medium confidence is where delete + log belongs, and timeouts or repeated-hit escalation should stay reserved for high-confidence lanes.

Allowlist and scope controls

Admins can carve out trusted roles, included or excluded scope, allowlisted domains, invite codes, phrases, solicitation carve-out channels, pack-specific exemptions, and built-in trusted pack entries so weak or educational context stays bounded. Global scope stays under Scope, pack-local exemptions live in the Rules panel, and /shield exemptions stays the quick slash fallback. Domain and invite allowlists stay scoped to Shield trusted-link policy, while phrase allowlists suppress only targeted promo or adult-solicitation text matches.

Focused admin lifecycle

Babblebox keeps the admin lane compact: one direct emergency lock lane plus follow-up, exclusions, logs, and truthful permission diagnostics. The remaining control surface is intentionally small and avoids sprawling server-wide control-plane work.

/lock channel, /lock remove, /lock settings, /timeout remove

Run Babblebox's compact emergency moderation lane.

Moderators who can manage channels or messages, timeout, kick, or ban members can lock or unlock normal text channels by default, admins can switch the lock lane to admins-only, Babblebox intentionally refuses category-synced channels, timed locks restore only the @everyone flags Babblebox actually changed, equivalent pre-locked channels can be tracked without reopening them later, and /timeout remove safely clears active member timeouts when moderator and bot hierarchy still allow it.
/shield panel

Open the panel-first Shield admin surface.

Use this first if you want the current state without memorizing command flags. Pick a pack, then Babblebox splits editing into Actions, Options, and Exemptions so only the relevant controls appear. The toggle controls live-message moderation; the bounded feature-surface lane stays on.
/shield module

Turn live Shield moderation on or off.

This is the direct slash fallback for the live moderation toggle. The bounded feature-surface safety lane still stays private and separate.
/shield escalation

Configure repeated-hit escalation and the global timeout fallback.

Use this when you want to tune delete_escalate thresholds or the global timeout profile without mixing those server-wide settings into pack-local rule changes.
/shield rules

Use the precise slash fallback for one pack's actions, relevant thresholds, and pack-specific timeout override.

Panel-first editing lives in /shield panel. Use this when you already know the pack and want to set Low/Medium/High actions, Anti-Spam or GIF thresholds, the GIF lane's one-member rate, low-text repeat gate, same-asset repeat, true channel-streak, and effective GIF-pressure settings with lightweight meaningful-text weighting, optional emote or capitals lanes, moderator anti-spam policy, adult solicitation handling, or that pack's dedicated timeout override without opening the panel.
/shield links

Choose Shield Default or Trusted Links Only mode for live messages.

Shield link policy is separate from Confessions link mode. Domain and invite allowlists act only as trusted-link policy exceptions, while malicious, impersonation, suspicious, or adult-domain intel still wins. The trusted-link lane can also keep its own dedicated timeout profile.
/shield trusted

Inspect the built-in trusted pack and turn specific built-in families or domains on or off for this server.

This makes Trusted Links Only auditable instead of opaque. Admin allowlists still stay separate from built-in trust, and hard-risk overrides still win.
/shield severe category, /shield severe term

Turn severe-harm categories on or off and manage bounded bundled or custom severe terms.

The severe pack is meant for explicit self-harm encouragement, exploitative sexual-abuse solicitation, eliminationist or dehumanizing hate, and severe slur abuse with quote, reporting, moderation, and rules-context suppressors, not general rudeness.
/shield logs

Choose the private mod-log channel, optional alert role, and calmer delivery policy.

Shield stays calmer when its evidence goes to one review lane instead of spraying public callouts. Global Adaptive vs Compact logging and Smart vs Never Ping delivery now apply cleanly across Shield packs, while one-user GIF floods group per offender and collective GIF pressure groups per channel so one takeover does not become its own alert flood.
/shield filters

Adjust included or excluded scope, trusted roles, and solicitation-only carve-out channels.

The channel carve-out only relaxes the optional solicitation detector; adult-domain, scam, and malicious-link protections still run there.
/shield exemptions

Exempt one member, role, or channel from one specific Shield pack.

This is the quick slash fallback when you already know the pack and target. The same pack-local exemptions now live directly inside the Rules panel so one bot-game channel can skip Anti-Spam, one media channel can skip GIF Flood / Media Pressure, or one trusted role can bypass one pack without weakening the rest of Shield.
/shield allowlist

Teach Shield which domains, invite codes, and phrases should get bounded policy carve-outs.

Domains and invites bypass only Trusted Links Only policy, while phrases suppress only targeted promo or adult-solicitation text matches. Malicious, suspicious, and adult-domain intel still wins.
/shield ai

Set Shield AI review scope for privacy, promo, scam, adult, and severe packs.

The support server has full Shield AI access by default, ordinary guilds stay off by default, and owner-only private policy decides which guilds and model tiers can actually use AI. Shield currently routes between gpt-5.4-nano, gpt-5.4-mini, and gpt-5.4, and diagnostics now report the resolved routing lane plus local readiness. Shield AI stays second-pass only, live-message-only, and only enriches moderator alerts after local Shield already flagged a message.
/admin panel

Open the interactive admin lifecycle surface.

Use this sectioned control surface to directly configure follow-up, exclusions, and logs. The overview quick-config row jumps straight into follow-up, exclusions, and logs, while /admin permissions stays close by as the targeted diagnostics fallback. Slash commands still stay as the precise fallback path.
/admin followup

Configure returned-after-ban follow-up roles.

This lane stays focused on clear return windows, auto-remove vs review handling, and compact review actions instead of building a giant moderation suite.
/admin logs, /admin exclusions, /admin permissions

Finish the focused admin lane with shared logs, trusted-role exclusions, and truthful permission diagnostics.

The panel now exposes direct editors for the common settings, while these slash commands remain the stronger precision fallback when you already know the exact target change.
FAQ

Common questions

Does Babblebox default to public or private replies?

It depends on the surface. Party lobbies, Daily cards, Buddy, and Profile usually lean public. Personal tools and setup flows usually stay private.

Do I need every feature turned on?

No. Babblebox is meant to work well when a server adopts only one lane first. You can use it just for games, just for utilities, just for identity surfaces, or add Shield live moderation only if you need it while the bounded feature-surface safety lane still protects the relevant private flows.

Does Shield automatically punish every match?

No. Shield can log, delete, timeout, or escalate depending on the pack and the configured confidence tier. Low-confidence lanes stay calm, Anti-Spam needs real corroboration before stronger actions fire, one-user spam and GIF floods can remove the matched burst, collective GIF pressure still needs real channel degradation and only removes the exact live GIF streak or newest contributing GIF posts that crossed the rule, mixed incidents can pair that shared cleanup with one member's personal GIF enforcement, and ordinary mixed chat is meant to stay safer than obvious floods.

Does Shield AI read my whole server?

No. Shield AI only reviews already-flagged live messages, uses a sanitized excerpt with compact metadata, does not decide punishment on its own, and stays owner-managed by default outside the support server.

Will Babblebox work well on Discord mobile?

That is the intent. Public cards are written to scan quickly, and the product tries to avoid oversized admin slabs or dense utility dumps.

Where should I ask for support or report an issue?

Use /support in Discord for the official links card, then jump to the support server for live help or GitHub if you want to report a bug or inspect the open-source code directly.
