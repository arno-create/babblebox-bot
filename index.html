Privacy policy

Babblebox is built to do the minimum necessary to provide its features. This section explains that in plain language.

How Babblebox handles data

Babblebox may process command input, game actions, DM replies needed for game flow, AFK status data, and the messages or attachments required to run features you actively use. It is designed around temporary runtime state rather than invasive tracking.

What may be processedCommand usage, temporary game state, player turns, DM responses used by the games, AFK status information, and the content needed to operate the feature a user chooses to use.
What is not the goalBabblebox is not built to sell personal data, build ad profiles, or act like a surveillance product. It is a game bot, not a data-mining machine.
Retention approachMost gameplay data is intended to be temporary and tied to the active session. Runtime state such as lobbies, timers, and AFK entries exists to keep the bot functioning and should not be treated as permanent storage for personal information.
DM usageSome games rely on direct messages for prompts, replies, or role information. If users disable DMs from server members, those features may not work correctly.
Sensitive informationDo not send passwords, payment details, legal identifiers, or other private information through the bot. Even responsible bots are not the right place for secrets.
Third-party servicesBabblebox may rely on hosting providers or Discord’s own platform infrastructure to operate. Those services have their own policies and technical limits.
ChangesAs the project evolves, this privacy section may be updated to reflect new features or operational changes. When in doubt, assume the bot should only be used for normal game-related interaction.
