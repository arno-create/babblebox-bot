Babblebox
the party game bot that actually feels alive.

Babblebox is built for the kind of moments people screenshot, quote later, and bring up again a week after the server call ends. It mixes voice-message chaos, social deduction, absurd storytelling, interactive views, and fast multiplayer tension into one polished Discord experience. This version also adds a modular backend, dual slash and bb! commands, and a quiet utility layer for Watch alerts, Later markers, Capture, Remind, and BRB without stressing a free-tier Render deployment.

Core commands, two ways.

Switch between slash and prefix examples. Core games and the new utility tools both support both routes.

/help Open the full game manual.
/ping Quick health check.
/play Open a lobby and host a game.
/stop Host/admin force stop.
/afk Set, schedule, or clear AFK.
/afkstatus View AFK state.
/chaoscard Cycle the current lobby twist.
/watch settings View your watch alert configuration.
/watch keyword add Add a watched keyword or phrase.
/later mark Save your place in the current channel.
/capture DM yourself a private snapshot of recent messages.
/remind set Create a one-time reminder.
/brb set Start a timed BRB notice.

Privacy policy

Babblebox is built to do the minimum necessary to provide its features. That applies to both the party games and the new personal utility tools.

How Babblebox handles data

Babblebox may process command input, game actions, DM replies needed for game flow, AFK status data, and the messages or attachments required to run features you actively use. It is designed around temporary runtime state rather than invasive tracking.

What may be processedCommand usage, temporary game state, player turns, DM responses used by the games, AFK status information, and content needed to operate the feature a user chooses to use.
What is not the goalBabblebox is not built to sell personal data, build ad profiles, or act like a surveillance product. It is a party game bot, not a data-mining machine.
Retention approachMost gameplay data is temporary and tied to the active session. Utility settings and reminders can use Postgres-backed storage for stronger continuity, with local JSON fallback for development or degraded startup cases. That still is not a promise of perfect permanence across outages, misconfiguration, or infrastructure loss.
DM usageSome games rely on direct messages for prompts, replies, or role information. If users disable DMs from server members, those features may not work correctly.
Sensitive informationDo not send passwords, payment details, legal identifiers, or other private information through the bot.
Third-party servicesBabblebox may rely on Discord, Render, and other infrastructure providers to operate. Those services have their own policies and limits.
