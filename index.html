The bot is now split into cogs and shared services, so game flow, AFK logic, commands, and the Flask keep-alive layer can evolve without turning back into a 3,000-line monolith.

Dual command system

Every core command now works as both a slash command and a prefix command. Use whichever style matches your server culture.

/help Open the manual and rules.
/ping Quick health check.
/play Create a lobby and choose a game.
/stop Host or admin stop.
/afk Enable, schedule, or clear AFK.
/afkstatus Show the current AFK record.
/chaoscard Cycle the active lobby twist.

Modular backend

The refactor moves the project away from a single-file bot and into a package layout that fits long-term maintenance and Render free-tier limits.

What moved where

  • cogs/ now owns commands and listeners.
  • game_engine.py keeps shared game state, views, timers, and flow logic.
  • bot.py handles startup, dictionary loading, and command sync.
  • web.py runs the lightweight Flask keep-alive server.

Why it fits Render

  • No polling database workers were added.
  • The keep-alive thread is daemonized for cleaner shutdown behavior.
  • The Word Bomb dictionary now caches to disk to avoid repeated full downloads.
  • Chaos Cards are stateless lobby modifiers, not long-running jobs.
